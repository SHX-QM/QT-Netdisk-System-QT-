数据表设计(sqlite3)
    用户信息表(usrInfo)
    字段      类型      约束条件      其他
    id        int      主键          自动增长
    name  varchar(32)  unique
    pwd   varchar(32)  not null
    online    int      default 0

    用户好友表(friend)
    字段      类型      约束条件      其他
    id        int      主键          外键
    friend    int      主键          外键

    字段是表的列；
    数据类型是字段存储的数据种类；
    主键是表的 “唯一标识列”，用来定位一条记录(比如id=1唯一对应 “张三” 这个用户)，特性：不能重复、不能为空；
    外键是表之间的 “关联桥梁”，保证字段值必须来自另一个表的主键(比如friend表的id必须是usrInfo表中存在的用户ID)；
    NOT NULL要求字段不能为空；
    自动增长:插入数据时，该字段不用手动赋值，数据库自动分配递增的数字(比如id从 1、2、3… 依次分配);
    varchar(32)：可变长度字符串，最多存32个字符

    usrInfo创建指令:create table usrInfo(id integer primary key autoincrement,name varchar(32) unique,pwd varchar(32),online integer default 0);
    friend创建指令:create table friend(id integer not null,friendId integer not null,primary key(id,friendId)); (primary key(id,friendId)是复合主键，保证好友关系只能插入一次，但视频中给出的这条指令不能实现外键约束，导致不属于usrInfo的id号也可插入friend)。
    friend表正确写法:
    第一步：开启SQLite外键约束（SQLite默认关闭，须手动开）
    pragma foreign_keys=ON;
    第二步：创建好友表
    create table friend(id integer not null,friendId integer not null,primary key(id,friendId),foreign key(id) references usrInfo(id),foreign key(friendId) references usrInfo(id));

	接着在构造函数中绑定信号和槽
	connect(&m_tcpSocket, &QTcpSocket::connected, this, &TcpClient::showConnect);    //connected：客户端与服务器端TCP连接成功后，由客户端自己的socket发出的信号

	最后连接服务器m_tcpSocket.connectToHost(QHostAddress(m_strIP),m_usPort);
	//发起异步Tcp连接，QHostAddress是Qt为IP地址量身定做的 “包装类”，把IP地址(不管是IPv4/IPv6、字符串/数值形式)封装成一个统一的对象

结构体大小计算
    遵循内存对齐规则:每个成员的偏移地址必须是(自身类型宽度)整数倍；结构体总大小必须是(最大成员类型宽度)整数倍
    示例1:struct Test1 
    {
        char a;    // 类型大小1字节
        short b;   // 类型大小2字节
        int c;     // 类型大小4字节
    };
    假设a   存储在第0字节
    则  b   存储在第2-3字节(b存储的起始位置必须是b类型大小的整数 倍，所以必须从第二字节存起)
    c   存储在第4-7字节
    该结构体最大成员长度是4，,总大小0-7共8字节是4的整数倍，无需填充空位，并非1+2+4=7
    示例2:struct DataBadOrder
    {
        char flag;          // 1字节
        long long fileSize; // 8字节
        int msgType;        // 4字节
        char name[5];       // 5字节
    }
    假设flag存储在第0字节
    则fileSize存储在8-15字节(同上面的b，1-7字节按规则要空出来)
    msgType存储在16-19字节
    name[5]存储在20-24字节
    此时结构体占据0-24位共25字节!=1+8+4+5
    由此可以知道即使相同的成员变量如果位置不同结构体大小也可能不同，因此设计结构体要考虑如何设计才能最大利用内存空间


strcpy strncpy memcpy   toStdString() c_str()  sprintf()   sscanf()
    strcpy:原型:char *strcpy(char *dest, const char *src);
    功能:从src拷贝以\0结尾的字符串到dest，包括\0
    strncpy:原型:char *strncpy(char *dest, const char *src, size_t n);
    功能:最多拷贝n个字符到dest；
    若src长度< n：剩余空间自动补\0；
    若src长度≥ n：只拷贝前n个字符，不补\0(需手动补0)
    memcpy:原型:void *memcpy(void *dest, const void *src, size_t n);
    功能:从src拷贝指定字节数n到dest，不看\0，严格按字节数拷贝

    toStdString():QString类的成员函数，作用是将Qt字符串(QString内部以UTF-16编码存储,每个字符占 2 字节)转换为C++标准库字符串(std::string是C++ 标准库字符串类型，内部以单字节字符数组存储，编码取决于操作系统，Windows默认为 GBK/GB2312，Linux/macOS默认为UTF-8)
    c_str():std::string类的成员函数，作用是返回指向 std::string内部字符数组的const char*指针

    sprintf()功能:把数字、字符等数据，按照格式 打印到字符数组里
    示例:sprintf(pdu->caData,"%s %lld",strFileNameUtf8.constData(),netFileSize);把strFileNameUtf8,netFileSize按照字符串类型 长整型格式写入pdu->caData
    sscanf()功能:从一个字符串里，按格式把数据提取出来
    示例:sscanf(pdu->caData,"%s %lld",caFileNameUtf8,&netFileSize);把pdu->caData中数据岸字符串类型 长整型格式读取到caFileNameUtf8,&netFileSize，&是为了更改引用的变量
    sprintf sscanf要连用，sprintf拼接的部分以空格间隔开，sscanf识别空格将数据分开读取出来

    服务器recvMsg处理逻辑:取出当前路径，设置过滤规则(排除.和..，分别代表当前文件夹本身和父目录)和排序规则后读取当前路径下的文件信息，将这些文件的名称大小等信息(可扩展，fileInfo读取到的不止这些信息)打包进respdu->caMsg部分发送过去
    pBook->updateFileList(pdu)处理逻辑:遍历读取到的文件信息，按照文件类型设置qt默认的图标文件并显示在m_pBookListW上

    槽函数Book::rename处理逻辑:选中文件，把文件旧名字 新名字 路径打包发送过去
    服务器recvMsg处理逻辑:获取客户端发送信息，拼接旧文件路径和新文件路径，调用dir.rename(strOldPath,strNewPath)覆盖旧文件名，将执行结果返回客户端

    客户端recvMsg处理逻辑:传入含有当前路径下文件信息的pdu执行刷新功能pBook->updateFileList(pdu)，并更新当前路径m_strCurPath，清空m_strEnterDir

上传文件
    点击上传文件按钮->m_pUploadPB发出&QPushButton::clicked信号->槽函数Book::uploadFile响应->服务器recvMsg接收并处理->定时器到时客户端Book::uploadFileData进行响应->服务器recvMsg接收上传文件数据->write()将服务器回复发送回客户端->客户端recvMsg接收信息并处理
    槽函数Book::uploadFile处理逻辑:获得选中文件的绝对路径并保存后提取文件名 文件大小 文件存储路径(当前路径)，发送给服务器后启动m_pTimer->start(1000);
    服务器recvMsg处理逻辑:获取客户端发送信息，拼接上传文件路径并设置服务器端m_file操作路径，之后修改参数m_bUpload=true;
                m_iTotal=fileSize;
                m_iRecved=0;
    定时器到时后Book::uploadFileData进行响应:停止定时器，循环传输目标文件数据，当ret=file.read(pBuffer,4096)只为0时停止传输
    服务器recvMsg接收上传文件数据处理逻辑:m_bUpload==true时将传输数据循环写入m_file绑定路径中，根据m_iTotal和m_iRecved大小关系回复上传结果
    客户端recvMsg处理逻辑:显示服务器回复并调用pBook->flush()刷新文件显示

    槽函数Book::uploadFile处理逻辑:获取选中的文件名，弹出窗口获取文件下载路径并保存，将当前路径和要下载的文件名发送到服务器
    服务器recvMsg处理逻辑:获取客户端发送信息，拼接要下载的文件路径后获取该文件大小，将要下载的文件名 文件大小回复给客户端后给服务器端m_file绑定要下载文件的路径并启动定时器m_pTimer->start(1000);
    客户端recvMsg处理逻辑:获取服务器发送信息，修改m_bDownload，给客户端m_file绑定文件下载路径
    定时器到时后MyTcpSocket::sendFileToClient进行响应:停止定时器，循环传输目标文件数据，当ret=file.read(pBuffer,4096)只为0时停止传输
    客户端recvMsg处理下载数据:m_bDownload==true时将传输数据循环写入客户端m_file绑定路径中，根据m_iTotal和m_iRecved大小关系显示下载结果
